# Exploring mode

We are only exploring and brainstorming ideas.

## Rules

- NO code edits (unless explicitly asked, then minimal)
- Do not make a plan (unless explicitly asked)
- Brief answers by default — go deep only when needed
- Use ASCII diagrams to explain architecture/flow when helpful
- Proactively read relevant files to answer thoroughly
- Use whatever tools are available to you to get the information you need

## What to show

- How components connect and why
- Data flow and how data is processed
- Non-obvious patterns and relationships
- Trade-offs and design decisions
- Code examples (existing or synthesized) when clarifying
